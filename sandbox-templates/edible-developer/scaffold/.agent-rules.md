# Agent Rules for Code Modifications

## Architecture: Three Layers

This project follows a strict three-layer architecture:

1. **Engine SDK** (`engine/sdk/`) — API client, types, mock data
2. **Adapter Layer** (`engine/`) — hooks, transforms, provider
3. **Presentation Layer** (`components/`, `pages/`) — UI components and pages

## YOU CAN MODIFY

- Any file in `components/` — change JSX, styling, layout, add new components
- Any file in `pages/` — rearrange page layouts, add new pages, add sections
- `tailwind.config.ts` — change theme colors, fonts, spacing
- `styles/` — any CSS or style-related files
- `public/` — assets, images
- `site.config.json` — branding values (name, colors)

## YOU MUST NOT MODIFY

- Any file in `engine/` — this is the booking engine integration layer
- `lib/query-client.ts` — TanStack Query configuration
- `package.json` dependencies (unless adding a UI-only library)

## COMPONENT CONTRACT

When modifying booking components, you MUST preserve:
- The component's props interface (the types it accepts)
- All hook calls at the top of the component
- The onSelect/onSubmit/onChange callback signatures

You CAN freely change:
- All JSX structure and nesting
- All Tailwind classes and styling
- Addition of new visual elements (badges, icons, dividers)
- Layout and responsive breakpoints
- Animation and transitions

## DATA FLOW

- Presentation components import display types and hooks from `@/engine`
- Components NEVER import directly from `@/engine/sdk`
- All data transformations happen in `engine/transforms/`
- All API calls happen in `engine/hooks/` via TanStack Query

## AVAILABLE HOOKS

- `useListingSearch(formState)` — search listings with filters
- `useListing(id)` — get a single listing by ID
- `useAvailability({ listingId, checkIn, checkOut, guests })` — check availability & pricing
- `useCreateReservation()` — mutation to create a reservation
- `useReviews(listingId)` — get reviews for a listing

## AVAILABLE DISPLAY TYPES

- `DisplayListing` — listing data for cards and detail pages
- `SearchFormState` — search form state (location, dates, guests, filters)
- `DisplayPricing` — formatted pricing breakdown
- `DisplayReservation` — reservation confirmation data
- `DisplayReview` — review data for display
- `AvailabilityData` — availability check result

## INSTALLING PACKAGES

Use `bun add <package>` for any new UI dependencies.
All shadcn/ui components are pre-installed and available in `components/ui/`.
